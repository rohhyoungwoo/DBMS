계정 생성 파일.

1. DBMS 총 복습
1) DB, DBMS
	DB : 데이터가 저장되는 저장공간
	DBMS : 데이터 베이스 관리 시스템, DB를 관리할 수 있는 구체적인 체계(시스템)
	=> 데이터베이스와 데이터 관리 시스템을 통해 데이터는 효율적으로 저장, 관리, 검색, 공유 될 수 있다

2) 특징
	일관성	: 여러 파일에 중복해서 저장되어있는 데이터를 하나로 모아 하나의 저장방식으로 종합해서 관리하는 것
	독립성	: 통합해서 저장한 데이터를 컴퓨터 운영체제와 별도의 데이터 관리 시스템에서 관리하는 것
	동시접속	: 여러 사용자 또는 여러 응용프로그램이 동시에 데이터를 읽고 쓸 수 있는 환경 지원하는 것
	공유	: 여러 사용자 또는 응용프로그램이 데이터를 공유하는 데 사용되는 것

3) DBMS 종류
	관계형 : 테이블 형식으로 데이터 저장, 테이블간의 관계를 설정하는 DBMS
	NOSQL : 관계형과는 다른 데이터 모델을 사용하는 데이터베이스 관리 시스템
	객체지향 : 객체지향 프로그래밍의 개념을 데이터 베이스에 적용하는 데이터베이스 관리 시스템
	분산	: 여러 컴퓨터 노드에 데이터를 분산하여 저장하고 처리하는 데이터베이스 관리 시스템

4) DDL, DML, DCL, TCL
	DDL : 데이터 정의 언어
		CREATE, ALTER, DROP, TRUNCATE
	DML : 데이터 조작 언어
		SELECT, INSERT, UPDATE, DELETE (중복, WHERE, 용량 O)
	DCL : 데이터 제어 언어
		GRANT, REVOKE
	TCL : 트랜잭션 제어 언어
		COMMIT, ROLLBACK

SELECT DISTINCT 컬럼명 AS 별칭
FROM 테이블 JOIN 테이블2
ON 조인조건 (컬럼 == 컬럼) //등가조인(완전 일치, PK와FK있을때), 비등가조인(특정 컬럼이 일치하지 않는 경우, 관계연산자 사용되었을때)
WHERE 검색조건
GROUP BY 그룹
HAVING 그룹에 대한 조건
ORDER BY 정렬 ASC / DESC

*SELECT 별칭은 ORDER BY 에서만 사용 가능

5) VIEW
	기존의 테이블은 그래도 두고 필요한 컬럼들 및 새로운 컬럼을 만든 가상 테이블
	실제 데이터가 저장되는 것은 아니지만 VIEW를 통해서 데이터를 관리할 수 있다

	- 독립성, 편리성, 보안성
	
6) 모델링과 정규화
	정규화 : 데이터 중복 최소화, 데이터 무결성을 향상시키는 방법
		이상현상을 제거하기 위한 작업(삽입이상, 삭제이상, 갱신이상)
	1차 : 모든 컬럼은 고유한 값을 가져야한다, 모든 행은 고유한 식별자(PK)를 가져야한다
	2차 : 컬럼은 기본키 전체에 종속되어야한다 (조합키만들어보면 알 수 있음)
	3차 : 기본키가 아닌 컬럼이 다른 컬럼을 결정하면 안된다 (우편번호, 주민등록번호, 성별 경우 테이블 분리)

	모델링
	요구사항 분석 -> 개념모델링 -> 논리모델링 -> 물리모델링 -> 구현
